## About Me

I am a security engineer from Hangzhou, China.

My work sits between **offensive security**, **secure engineering**, and **security automation**.
I enjoy turning security experience into tools, systems, and repeatable workflows.

Over the years, I have worked on:

* Web security, penetration testing, and vulnerability research
* Source code auditing and static analysis
* Enterprise security architecture and security operations
* AI-assisted security analysis and multi-agent security workflows
* Web3 / Solana infrastructure, reverse engineering, and trading system research

I like practical security work: less buzzword, more evidence, more automation, more reproducible results.

## Philosophy

> Security is not only about finding bugs.
> It is about building systems that make risks visible, explainable, and controllable.

I believe good security engineering should be:

* **Practical**: useful in real environments
* **Evidence-based**: every conclusion should be traceable
* **Automated**: repeated work should become tools
* **Readable**: security knowledge should be easy to share
* **Hacker-minded**: always curious, always questioning
